# P-20260526-002: ai-superpower-dev V4 — Web Audit Replay UI + Replay API

## 1. 目标

在 Web UI 的 Audit Log 页面增加 Replay 操作能力：用户可以选中某条记录，点击 "Undo" 按钮回滚该操作。

## 2. 现状分析

已有能力：
- ✅ audit.log 为 JSON Lines 格式（字段级变更记录）
- ✅ `replay.py` CLI 支持 `--undo <entity_id>`、`--dry-run`、`--from`、`--last`
- ✅ `test_replay.py` 22 个测试全部通过
- ✅ `GET /api/audit` 分页 API 已存在

缺失能力：
- ❌ Web UI 无 Replay 操作入口
- ❌ 无 Replay API 端点（CLI replay 无法从 Web 触发）

## 3. 技术方案

### 3.1 新增 API 端点

```python
POST /api/replay/undo
Body: {"entity": "project"|"proposal", "id": "PRJ-xxx"|"P-xxx"}
Response: {"success": true, "message": "...", "entry": {...}}
```

### 3.2 Web UI 增强（audit.html）

- 每条 audit 日志显示"Undo"按钮
- 点击后 POST `/api/replay/undo` 并显示结果
- dry-run 模式下仅预览（显示提示）

### 3.3 安全考虑

- Undo 操作需要 API Key 认证（复用现有 `X-API-Key` header）
- 危险操作（DELETE undo）返回警告但不阻止

## 4. 测试用例（TDD）
