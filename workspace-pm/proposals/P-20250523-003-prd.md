# P-20250523-003: ai-superpower 增强需求

## 1. Audit Log 增强 + Replay

### 目标
将 audit.log 升级为完整的字段级变更日志，支持 JSON 格式和 replay 回滚。

### 设计

**日志格式：JSON Lines（每行一个 JSON）**

```json
{"ts":"2026-05-23T15:00:00+08:00","op":"UPDATE","entity":"proposal","id":"P-20250523-001","field":"status","old":"intake","new":"in_review","actor":"sha256_of_api_key","checksum_after":"abc123"}
```

**操作类型：** `CREATE` / `UPDATE` / `DELETE`

**字段说明：**
- `ts`: ISO8601 时间戳（含时区）
- `op`: 操作类型
- `entity`: `project` 或 `proposal`
- `id`: 实体 ID
- `field`: 变更字段名（UPDATE 时有，CREATE/DELETE 时为 null）
- `old`: 旧值（UPDATE/DELETE 时有，CREATE 时为 null）
- `new`: 新值（CREATE/UPDATE 时有，DELETE 时为 null）
- `actor`: API Key 的 SHA256 前8位（可识别操作者）
- `checksum_after`: 操作后文件 SHA256

**Storage 层变更：**
- `create_project` / `create_proposal`: 写入后计算全文件 checksum
- `update_project` / `update_proposal`: 逐字段比较，生成每字段一条 audit 日志；写入后计算 checksum
- `delete_project` / `delete_proposal`: 记录删除前的完整数据

**Replay 命令：**
```bash
ai-superpower replay --from 2026-05-23T00:00:00+08:00  # 从指定时间回放
ai-superpower replay --last N                             # 回放最近 N 条
ai-superpower replay --dry-run                            # 仅预览不执行
ai-superpower replay --undo P-20250523-001                # 撤销指定实体的最新操作
```

## 2. Web UI

### 目标
提供浏览器可访问的 HTML 界面，替代或补充 TUI。

### 技术方案
- FastAPI + Jinja2 模板
- 静态文件（CSS/JS）从 `/static` 路径服务
- 前端 JS 通过 Unix socket 发 API 请求（复用 `APIClient` 逻辑，浏览器端用 `fetch` 通过代理转发）

**注意：** 浏览器无法直接连接 Unix socket，需要在 server.py 增加一个 HTTP 代理端点：
```
GET /api-proxy/<path>  →  转发到 Unix socket
```
或者 Web UI 走与 API Server 不同的端口（同一进程监听两个 socket）。

**简化方案：Server 同时监听 HTTP 端口（如 8080），Web UI 走 HTTP，CLI 走 Unix socket。**

### 路由设计

| 路径 | 说明 |
|------|------|
| `GET /` | 首页仪表盘 |
| `GET /projects` | 项目列表 |
| `GET /projects/<id>` | 项目详情 |
| `POST /projects` | 创建项目 |
| `PUT /projects/<id>` | 更新项目 |
| `GET /proposals` | 提案列表 |
| `GET /proposals/<id>` | 提案详情 |
| `POST /proposals` | 创建提案 |
| `PUT /proposals/<id>` | 更新提案 |
| `GET /audit` | 审计日志查看器 |
| `GET /settings` | 设置页（显示 API Key） |

## 3. 定时备份

### 目标
定时将 `db/` 目录完整备份，支持本地和远程（Git）。

### 配置项
```toml
[backup]
enabled = true
frequency = "1h"        # 1h / 6h / 1d
max_copies = 48         # 最多保留份数
local_path = "/home/hermes/ai-superpower/backups"
remote_repo = ""        # git@github.com:YeLuo45/ai-superpower-backup.git
remote_branch = "backup"
api_key = ""            # 用于 git push 的 HTTPS token
```

### 备份流程
1. `cp -r db db_backup_<timestamp>`
2. 压缩为 `db_backup_<timestamp>.tar.gz`（可选）
3. 保留最近 N 份，删除超出部分
4. 如果配置了 `remote_repo`，则 `git add + commit + push` 到远程

### 定时实现
- 使用 `APScheduler`（内存中调度，不依赖 cron）
- 在 FastAPI server 启动时注册调度任务
- 或独立 cron 进程调用 CLI 命令：`ai-superpower backup`

### CLI 命令
```bash
ai-superpower backup           # 立即执行一次备份
ai-superpower backup --list    # 列出已有备份
ai-superpower backup --restore db_backup_20260523_150000  # 恢复
```

### 备份文件命名
`db_backup_YYYYMMDD_HHMMSS/`
