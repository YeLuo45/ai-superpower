# P-20260525-001: ai-superpower-dev V5 — 同步配置 + 定时能力 + GitHub Pages 部署

## 1. 目标

1. 在 Settings 页面增加同步配置 UI（目标仓库、API Key、同步频率）
2. 增加定时同步能力（APScheduler）
3. 将本地 db 数据同步到 ai-superpower 的 GitHub Pages

## 2. 现状分析

### 已有能力
- ✅ `sync.py` — 同步 proposals 到 prj-proposals-manager（手动触发）
- ✅ `sync_gh_pages.py` — 同步 proposals + projects 到 GitHub Pages（手动触发）
- ✅ `POST /api/sync/export` — 触发 GitHub Pages 同步
- ✅ `POST /api/sync/push` — 触发 prj-proposals-manager 同步
- ✅ `GET /api/sync/config` — 读取同步配置
- ✅ `PUT /api/sync/config` — 更新同步配置

### 缺失能力
- ❌ Settings 页面无同步配置 UI
- ❌ 无定时同步（APScheduler）
- ❌ `data_dir` 需要指向 ai-superpower-dev 的 db 而非 ai-superpower 的

## 3. 技术方案

### 3.1 配置字段（config.toml 新增）

```toml
[sync]
enabled = true
frequency = "1h"          # 1h / 6h / 12h / 1d
github_pages_repo = "YeLuo45/ai-superpower"   # GitHub Pages 目标仓库
prj_repo = "YeLuo45/prj-proposals-manager"   # prj-proposals-manager 目标仓库
api_key = ""              # GitHub PAT（两个仓库用同一个）
```

### 3.2 Settings 页面 UI

在 Backup 区块下方增加 **Sync Configuration** 区块：
- GitHub PAT（密码输入，可显示）
- GitHub Pages 目标仓库
- prj-proposals-manager 目标仓库
- 同步频率下拉（关闭 / 1h / 6h / 12h / 1d）
- 立即同步按钮（手动触发）
- 上次同步时间显示

### 3.3 定时同步（APScheduler）

```python
from apscheduler.schedulers.background import BackgroundScheduler

# 在 server 启动时注册
scheduler = BackgroundScheduler()
scheduler.add_job(
    lambda: sync_all(config),
    'interval',
    hours=frequency_hours,
    id='sync_job'
)
```

### 3.4 API 端点

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/api/sync/config` | 读取同步配置 |
| PUT | `/api/sync/config` | 更新配置（含频率） |
| POST | `/api/sync/export` | 立即同步到 GitHub Pages |
| POST | `/api/sync/push` | 立即同步到 prj-proposals-manager |
| GET | `/api/sync/status` | 定时任务状态 |
