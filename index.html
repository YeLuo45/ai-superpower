ai-superpower

提案系统 API 引擎 — projects.csv 和 proposals.csv 所有变更的唯一入口

FastAPI + CSVStorage + JSON Audit

背景问题

为什么需要这个系统？

直接编辑 CSV（脚本、execute_code、手动 patch）会绕过校验、污染枚举字段、破坏引用完整性。ai-superpower 通过架构设计，使 API 成为唯一写入路径。

变更方式风险
直接 patch CSV绕过校验、污染枚举字段、破坏引用完整性
通过 API 写入Pydantic校验 + 状态机 + flock锁 + SHA256审计

核心防护机制

机制作用
API 唯一写入路径所有数据变更必须经过 API
Pydantic 校验ID 格式、枚举值、字符串长度、必填字段
状态机转换校验intake→clarifying→prd_pending_confirmation→...→deployed→delivered
flock 文件锁读并发、写串行化
JSON 审计日志字段级变更追踪 + 精确回滚
引用完整性project_id 存在性检查 + 级联删除保护
Unix Socket 传输不暴露网络端口
API Key 认证每次请求带 X-API-Key Header

状态机

intake → clarifying → prd_pending_confirmation → approved_for_dev ↓ in_tdd_test ←────────────────────── in_dev ↓ ↓ in_test_acceptance ←──────────────── needs_revision ↓ ↓ accepted test_failed ↓ deployed → delivered

API 端点

方法路径说明
GET/health健康检查（无需认证）
POST/projects创建项目
GET/projects列出项目（分页）
GET/projects/{id}获取单个项目
PUT/projects/{id}更新项目
DELETE/projects/{id}删除项目
POST/proposals创建提案
GET/proposals列出提案（分页+过滤）
GET/proposals/{id}获取单个提案
PUT/proposals/{id}/status更新状态（状态机校验）
PUT/proposals/{id}/fields更新字段
DELETE/proposals/{id}删除提案
POST/validate干跑校验
GET/audit查询审计日志
POST/replay回滚操作（dry-run/undo）
POST/backup立即备份

Web UI

启动 ai-superpower run 后访问 http://localhost:8000：

ai-superpower run # 访问 http://localhost:8000 # - /web/projects — 项目列表 + 新建/编辑 # - /web/proposals — 提案列表 + 筛选 + 新建 # - /web/audit — 审计日志时间线 # - /web/settings — 配置页面

定时备份

配置 ~/.ai-superpower/config.toml 启用：

[backup] enabled = true frequency = "1h" # 1h / 6h / 1d max_copies = 48 local_path = "/path/to/backups" remote_repo = "" # git remote for push remote_branch = "backup"

安装

# 从源码安装 cd ai-superpower pip install -e . --break-system-packages # 或使用安装脚本 bash deploy/install.sh

CLI 命令

# 启动服务器 ai-superpower run # 项目 ai-superpower project create --name "我的项目" ai-superpower project list ai-superpower project get PRJ-20250523-001 # 提案 ai-superpower proposal create --title "新功能" --owner alice --project-id PRJ-20250523-001 --stage ideation ai-superpower proposal list ai-superpower proposal get P-20250523-001 ai-superpower proposal update-status P-20250523-001 --status clarifying # 工具 ai-superpower validate --data '{"project_id":"PRJ-20250523-001","stage":"ideation"}' ai-superpower audit --page 1 --page-size 100 ai-superpower replay --last 10 --dry-run ai-superpower replay --undo P-20250523-001 ai-superpower backup ai-superpower sync-to-index
